Includes the resident-kernel optimizer loop (_optimizer_kernel_loop): one
kernel call per fix, the kernel pulling each next point from the host
optimizer generator over sync RPC and applying it device-side, so ANY host
strategy (grid, coordinate descent, Bayesian optimization) pays one
compile+upload — no recompilation between points. The kernel-driven DAG fix
(prepare_kernel_fix() + fix_state_kernel()) fixes an entire calibration
dependency tree from within a single kernel. Also the live optimizer trace
(calibrations.optimizer dataset): every optimizer point is broadcast as it is
measured so an applet can watch the scan happen (see
repository/lib/applets/qbutler_optimizer_applet.py).
