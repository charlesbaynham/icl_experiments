Precompiled per-node calibration architecture:
- precompile.py: PrecompilePool — a background worker thread that precompiles
  node-keyed kernel callables once (~16 s each) so the host walk and the
  escape/re-enter loop deploy a ready artifact (~0.24 s) instead of recompiling
  on every kernel touch. Thread-safe against a running kernel only for
  subkernel-free kernels (CI guards this).
- client.py: CalibratedExpFragment / make_calibrated_experiment /
  run_calibrated / recalibrate_if_needed — the physicist-facing base class.
  A @kernel run_once calls recalibrate_if_needed() at a scan-point boundary;
  if a dependency drifted it raises CalibrationEscape, the host runs the DAG
  fix walk (pooled kernels) and re-enters the precompiled main kernel from the
  top (attribute_writeback=False, so first-run guards re-trigger — deliberate).
- calibration.py: the fused DAG-kernel machinery (__init_subclass__ auto-swap,
  prepare_kernel_ops, _fsk_*/_csk_* drivers, check_state_kernel/fix_state_kernel)
  is REMOVED. The host check_state/fix_state walk drives per-node precompiled
  kernels through seed_precompile_pool / _seed_own_precompile hooks. The
  per-node resident optimizer loop (_optimizer_kernel_loop) is retained: one
  kernel call per fix, host optimizer streaming points over RPC. CalibrationEscape
  is defined here (module-level, so it crosses the kernel→host boundary by class
  identity). The live optimizer trace (calibrations.optimizer dataset) is kept.
